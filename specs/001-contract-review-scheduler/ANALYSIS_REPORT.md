## 潛在風險與建議

### 高優先級建議

**建議 1: 補充前端無障礙性 (Accessibility) 測試任務**

- **問題**: Constitution 原則 III 要求 WCAG 2.1 AA 合規，但 tasks.md 中無明確測試任務
- **影響**: 前端開發完成後無法驗證無障礙性
- **建議動作**: 
  - 在 Phase 7 中補充任務：`T072b [P] [US1-US4] 進行 WCAG 2.1 AA 無障礙性測試 frontend/...`，使用 axe DevTools 或 WAVE 工具
  - 或將無障礙性檢查整合到 T070-T072 中

**建議 2: 確認前端文件語言標準**

- **問題**: Constitution 原則 VI 要求所有文件使用繁體中文，但 T076-T078 未明確確認
- **影響**: 前端部署文件可能使用英文
- **建議動作**:
  ```markdown
  修改 T077: [...] 編寫部署指南 `frontend/DEPLOYMENT_GUIDE.md` (繁體中文)
  修改 T078: [...] 編寫環境變數配置範本 `frontend/.env.example` (繁體中文註釋)
  ```

### 中優先級建議

**建議 3: 補充預約變更流程**

- **問題**: spec.md L45 列出「預約變更通知」，但使用者流程未明確定義變更如何執行
- **影響**: 開發者可能對變更流程有不同理解
- **建議動作**: 在下次 spec 審查時補充「預約變更」場景或在 US1 中詳細說明

**建議 4: 澄清郵件送達率責任**

- **問題**: spec.md L69 要求 99.9% 郵件送達，但此依賴公司 SMTP 伺服器
- **影響**: 應用層無法完全控制此指標
- **建議動作**: 在 plan.md 中補充說明責任邊界，應用層可控部分為「3 次重試機制」，總體送達依賴 SMTP

**建議 5: 優化前端測試覆蓋**

- **問題**: 前端測試任務 (T070-T072) 未明確涵蓋所有 8+ 前端元件
- **影響**: 某些元件可能缺少測試
- **建議動作**: 在 T070 中補充：「涵蓋 LoginPage, AppointmentForm, CalendarView, AppointmentList, AppointmentDetailPanel, LeaveScheduleForm, DelegateReviewerSelect, DelegatConfirmDialog」

### 低優先級建議

**建議 6: 補充性能基準詳細測試**

- **問題**: T069 性能測試較籠統，未明確說明測試方法 (負載測試工具、測試環境配置)
- **影響**: 可能難以重現或驗證
- **建議動作**: 補充 T069 細節：「使用 Apache JMeter 或 k6 進行負載測試，1000 並行用戶...」

**建議 7: 補充部署監控指標**

- **問題**: NFR-006 「99.5% 系統可用性」無法通過應用層測試
- **影響**: 部署後無法驗證
- **建議動作**: 補充 T075 檢查清單項目：「配置監控告警 (System availability, API response time, Error rate)」
